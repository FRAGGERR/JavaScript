  • Variable Rules
  1. Variable names are case sensitive; "a" & "A" is different.
  1. Only letters, digits, underscore(_) and $ is allowed. (not even space)
  1. Only a letter, underscore(_) or $ should be 1st character.
  1. Reserved words cannot be variable names.

The actual way to define the variable

let, const & var

  • var : Variable can be re-declared & updated. A global scope variable.
  • let : Variable cannot be re-declared but can be updated. A block scope variable.
  • const: Variable cannot be re-declared or updated. A block scope variable.

let name = "John"; // Represents Textual data

// Number

// Boolean
let isStudent = true; // Represents Boolean :  true or false values

// Null

// Undefined

To explain BigInt and Symbol in JavaScript, let's take a look at the following code:

// BigInt
const bigNumber = 1234567890123456789012345678901234567890n;
console.log(bigNumber); // Output: 1234567890123456789012345678901234567890n
console.log(typeof bigNumber); // Output: bigint

// Symbol
const sym1 = Symbol("sym1");
const sym2 = Symbol("sym2");

console.log(sym1); // Output: Symbol(sym1)
console.log(sym2); // Output: Symbol(sym2)
console.log(typeof sym1); // Output: symbol

In the code above, we demonstrate the use of BigInt and Symbol in JavaScript.

  • BigInt is a numeric data type introduced in JavaScript to represent integers of arbitrary precision. It allows you to work with integers beyond the safe integer limit of Number type. To create a BigInt, append n to the end of an integer literal or use the BigInt() function.
  • Symbol is a unique and immutable data type introduced in JavaScript. It is often used to create unique property keys in objects or to define unique constants. Each Symbol value is unique and cannot be equal to any other value, including other Symbol values.

Please note that BigInt and Symbol are relatively new additions to JavaScript and may not be supported in older browsers. Make sure to check the browser compatibility before using them in your code.
