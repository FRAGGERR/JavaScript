Functions in JS

In programming, a function is a self-contained block of code designed to perform a specific task or a set of tasks. Functions provide a way to structure code, promote reusability, and enable modular programming. They allow you to define a set of instructions that can be executed when the function is called or invoked.

1. Function Declaration:

Functions in JavaScript are blocks of code designed to perform a specific task. They are defined using the function keyword.

function greet() {
    console.log("Hello, World!");
}

2. Function Expression:

Functions can also be assigned to variables, creating function expressions.

let greet = function() {
    console.log("Hello, World!");
};

3. Function Invocation:

Functions are executed or invoked using parentheses ().

greet(); // Calling the greet function

4. Parameters and Arguments:

Functions can receive parameters, which act as variables within the function. Arguments are the values passed to the function when it is invoked.

function greet(name) {
    console.log("Hello, " + name + "!");
}

greet("Alice"); // Output: Hello, Alice!

5. Return Statement:

Functions can return a value using the return statement. The function stops executing once the return statement is encountered.

function add(a, b) {
    return a + b;
}

let result = add(2, 3); // Result: 5

6. Scope:

Variables declared inside a function are local to that function (function scope). Variables declared outside any function are global.

let globalVar = "I am global";

function exampleFunction() {
    let localVar = "I am local";
    console.log(globalVar); // Accessing global variable
    console.log(localVar); // Accessing local variable
}

exampleFunction();

7. Function Hoisting:

In JavaScript, function declarations are hoisted to the top of their containing scope. This means you can call a function before it appears in the code.

hoistedFunction(); // This works even though the function is called before its declaration

function hoistedFunction() {
    console.log("I am hoisted!");
}

8. Anonymous Functions and Arrow Functions:

JavaScript supports anonymous functions and arrow functions, which are concise ways of defining functions.

// Anonymous Function
let anonymousFunc = function() {
    console.log("I am anonymous!");
};

// Arrow Function
let arrowFunc = () => {
    console.log("I am an arrow function!");
};

9. Callback Functions:

Functions can be passed as arguments to other functions, creating callback functions.

function doSomething(callback) {
    console.log("Doing something...");
    callback();
}

function callbackFunction() {
    console.log("Callback function executed!");
}

doSomething(callbackFunction);

10. Immediately Invoked Function Expressions (IIFE):

IIFE is a pattern where a function is defined and immediately invoked.

(function() {
    console.log("I am an IIFE!");
})();

Arrow Functions:

Arrow Function Syntax:

Arrow functions provide a concise syntax for writing functions. The basic syntax is:

// Traditional Function
function add(a, b) {
    return a + b;
}

// Arrow Function
const addArrow = (a, b) => a + b;
//Multiplication arrow functions 
const mult = (a, b) => a * b;

let a = 10;
let b = 20;

console.log(mult(a, b)); 
//Addition arrow function 
const add = (a, b) => {
    return a + b;
}
console.log(add(2, 5));
